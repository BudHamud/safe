# 🏦 Safed - Financial Management System V2

**Safed** is a comprehensive expense and income management application designed with a premium, minimalist, and functional aesthetic. It provides total control over your finances with real-time multi-currency support and an optimized user experience for both web and mobile devices (Android).

## 📊 Key Features

- **Multi-Currency Management:** Native support for **ILS, USD, ARS, and EUR**. Automatic conversion and balance visualization in your preferred base currency with live exchange rates.
- **Category Discovery:** An intelligent system that identifies your categories from transaction history, featuring custom icon support and total personalization.
- **Travel Mode / New Account:** Isolate specific movements and balances for trips or specific periods without affecting your main historical data.
- **Premium Interface:** Dark mode design inspired by modern trends, featuring *glassmorphism* effects, micro-animations, and carefully curated typography (Outfit & Inter).
- **Data Security:** Card digit masking for credit cards and secure transaction handling.
- **Statistical Dashboard:** Clear visualization of fixed vs. variable expenses, monthly goal tracking, and total balance overview.
- **Multi-Platform:** Works as a Web application (Next.js) and as a Native Android app (Capacitor).

## 🛠️ Tech Stack

- **Frontend:** [Next.js 16](https://nextjs.org/) (App Router), React 19, Vanilla CSS.
- **Database:** [PostgreSQL](https://www.postgresql.org/) (connected via Prisma ORM).
- **Web Deployment:** [Vercel](https://vercel.com/) (Frontend) & [Railway](https://railway.app/) (Database).
- **Mobile:** [Capacitor](https://capacitorjs.com/) for Android packaging.
- **External APIs:** Exchangerate-API and DolarAPI for live currency quotes.

## 🚀 Local Setup

1. **Clone the repository and install dependencies:**
   ```bash
   npm install
   ```

2. **Configure environment variables:**
   Create a `.env` file in the root directory with the variables used by the web app:
   ```env
   DATABASE_URL="postgresql://user:password@host:port/dbname"
   NEXT_PUBLIC_SUPABASE_URL="https://your-project.supabase.co"
   SUPABASE_SERVICE_ROLE_KEY="your-supabase-service-role-key"

   # Optional
   DIRECT_URL="postgresql://user:password@host:port/dbname"
   GROQ_API_KEY="your-groq-api-key"
   ```

   Notes:
   - `DATABASE_URL` is required by Prisma.
   - `NEXT_PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` are required by the server-side auth and data routes.
   - `GROQ_API_KEY` is only required if receipt scanning remains enabled.
   - `DIRECT_URL` is optional for this app's web deploy, but can be useful for Prisma admin or migration workflows.

3. **Sync the database schema:**
   ```bash
   npx prisma db push
   ```

4. **Run in development mode:**
   ```bash
   npm run dev
   ```

## 🌐 Deployment

### 1. Database (Railway)
- Create a project on Railway and add a PostgreSQL service.
- Use the `DATABASE_PUBLIC_URL` for the initial synchronization (`db push`).

### 2. Web (Vercel)
- Connect your GitHub repository to Vercel.
- Add these environment variables in Vercel:
   - `DATABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `SUPABASE_SERVICE_ROLE_KEY`
   - `GROQ_API_KEY` only if you want receipt scanning enabled in production
   - `DIRECT_URL` only if your database workflow requires it
- The build command will automatically run `prisma generate` due to the `postinstall` script.

### 3. Mobile (Android)
- Use the production config before generating distributable artifacts.
- Execute:
  ```bash
   npm run build
   npm run cap:prod
   npm run aab:prod
  ```
- For Play Store submission, sign the generated release bundle from Android Studio or your CI signing pipeline.

### 4. Android release signing
- The Android release build now supports signing from either `android/keystore.properties` or environment variables.
- Copy `android/keystore.properties.example` to `android/keystore.properties` and fill in your real values, or export these environment variables:
   - `SAFED_UPLOAD_STORE_FILE`
   - `SAFED_UPLOAD_STORE_PASSWORD`
   - `SAFED_UPLOAD_KEY_ALIAS`
   - `SAFED_UPLOAD_KEY_PASSWORD`
- If those values are missing, Gradle still builds an unsigned release bundle for validation only.
- For Play upload, you must provide a real upload key so `app-release.aab` is signed.
- Full step-by-step release instructions are in `reports/PLAY_RELEASE_STEPS.md`.

## 🏗️ Project Structure

- `src/app`: Application pages and routing (Next.js App Router).
- `src/app/api`: Backend routes for transactions, users, and categories.
- `src/app/components`: Modular UI components.
- `src/context`: Global state (AppContext) and synchronization logic.
- `prisma`: Data schema and database client configuration.
- `android`: Native project files generated by Capacitor.
